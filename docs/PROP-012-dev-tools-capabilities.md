# PROP-012: Dev-tools capabilities для релевантных ответов

## Статус
Принято

## Контекст
Модель в части запросов переориентируется на базовые файловые утилиты (`cat`, `find`, `grep`) даже в доменных сценариях разработчика (Docker, Git, systemd и другие рабочие инструменты).

## Решение
- Добавить отдельный список обнаруженных dev/devops-инструментов в capabilities.
- Передавать этот список в prompt как отдельный блок, не смешивая с policy-заменами baseline→modern.
- Явно указать в prompt: для доменных запросов выбирать профильный инструмент (`docker`, `git`, `systemctl`, `journalctl`, `kubectl`, и т.д.), а не подменять задачу файловыми командами.

## Последствия
- Плюсы: выше вероятность корректных ответов для реальных разработческих задач.
- Плюсы: сохраняется текущая strict/prefer-policy для известных замен (`grep->rg`, `find->fd` и т.д.).
- Минусы: capability-блок становится длиннее, что немного увеличивает токены в prompt.
