# PROP-014: Контекст рабочей директории в prompt

## Статус
Принято

## Контекст
Для терминальных задач важен локальный контекст: где находится пользователь и какие файлы/папки лежат рядом. Без этого модель чаще предлагает слишком общие или менее релевантные команды.

## Решение
- Добавить в capabilities путь текущей рабочей директории (`cwd_path`).
- Добавить в capabilities список элементов текущей директории (`cwd_entries`) с безопасным лимитом.
- Передавать эти данные в prompt как отдельный блок контекста.
- При большом количестве элементов отмечать усечение списка флагом `cwd_entries_truncated`.

## Последствия
- Плюсы: выше релевантность команд для текущего проекта и структуры файлов.
- Плюсы: меньше лишних уточняющих шагов для типовых локальных задач.
- Минусы: prompt становится длиннее; требуется ограничение размера списка.
