# PROP-013: Пользовательский приоритет инструментов через `--tools`

## Статус
Принято

## Контекст
Пользователь может точно знать, какие инструменты хочет использовать (`docker`, `git`, `rg` и т.д.), но не помнить конкретные флаги и синтаксис команд. Базовый prompt без явного user-priority не всегда стабильно отдает предпочтение нужным утилитам.

## Решение
- Добавить CLI-флаг `--tools` со списком инструментов через запятую.
- Нормализовать список (`lowercase`, удаление дублей и пустых значений).
- Прокинуть список приоритетных инструментов в prompt как отдельный контекстный блок.
- В prompt явно инструктировать модель: при уместности задачи сначала использовать инструменты из `--tools`.

## Последствия
- Плюсы: лучшее попадание в ожидаемый стек пользователя без ручного переформулирования запроса.
- Плюсы: сохраняется текущая policy-механика (`prefer/strict`) и валидация команд.
- Минусы: добавляется еще один управляющий флаг CLI и немного усложняется prompt.
