# PROP-004: UX-поведение CLI и completion

## Статус
Принято

## Контекст
Требовалось согласованное поведение CLI: минимальный вывод, безопасное подтверждение, reasoning-режим и удобство вызова.

## Решение

- В обычном режиме показывать только команду и prompt `[y/N]:`.
- В `-r` режиме не требовать подтверждения и не выполнять команду.
- Добавить `--dry-run` для проверки без исполнения.
- Добавить `--print-zsh-completion` и генерацию completion-скрипта.

## Последствия

- Плюсы: предсказуемый UX в терминале.
- Плюсы: проще встраивать в рабочий shell-процесс.
- Минусы: больше флагов и сценариев для поддержки/тестирования.
