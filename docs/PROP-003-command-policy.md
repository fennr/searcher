# PROP-003: Политика команд prefer/strict

## Статус
Принято

## Контекст
Модель иногда возвращала команды, не подходящие под желаемую policy использования modern-утилит.

## Решение
Ввести формализованную политику и пост-валидацию:

- `--prefer-modern` (по умолчанию): приоритет modern, разрешен fallback.
- `--strict-modern`: для известных замен требуется modern-инструмент.
- Второй проход “repair” при нарушении policy или невалидном формате команды.

## Последствия

- Плюсы: стабильнее поведение и управляемость для продвинутых пользователей.
- Плюсы: в strict-режиме явные ошибки вместо неявных отклонений.
- Минусы: логика валидации/ремонта стала сложнее.
