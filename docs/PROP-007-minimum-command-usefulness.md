# PROP-007: Минимальная полезность командного ответа

## Статус
Принято

## Контекст
Модель может вернуть синтаксически валидную, но практически бесполезную команду, например одиночный токен `bat` без аргументов.

## Решение
Добавить проверку минимальной полезности в командный policy-слой:

- Если ответ состоит только из имени известной утилиты без аргументов, считать его непригодным.
- В таком случае запускать шаг `repair` для получения полноценной команды.

## Последствия

- Плюсы: меньше бессодержательных ответов в обычном режиме.
- Плюсы: выше вероятность получить команду, которую можно сразу выполнить.
- Минусы: в редких случаях дополнительный запрос к модели для восстановления.
