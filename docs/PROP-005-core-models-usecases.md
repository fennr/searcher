# PROP-005: Разделение на core, models и use cases

## Статус
Принято

## Контекст
Текущая кодовая база стала сложнее и требует более явного разделения ответственности, строгой типизации и единообразной документации функций.

## Решение
Принять трехслойную структуру:

- `searcher/models` — типовые модели на базе `TypedDict`.
- `searcher/core` — чистая инфраструктурная и алгоритмическая логика.
- `searcher/use_cases` — прикладные сценарии и orchestration.

Дополнительно:

- Для прикладного кода включить подход совместимый с `mypy --strict`.
- Все функции снабжать docstrings.
- Не использовать комментарии внутри функций; сложность снижать за счет декомпозиции.

## Последствия

- Плюсы: выше поддерживаемость, проще развитие и ревью изменений.
- Плюсы: меньше неявных зависимостей между слоями.
- Плюсы: более надежная типизация JSON-структур и CLI-сценариев.
- Минусы: больше файлов и интерфейсных адаптеров.
