# PROP-010: Markdown-рендеринг в reasoning-режиме

## Статус
Принято

## Контекст
Ответы в `-r` режиме содержат Markdown (кодовые блоки, списки, выделение), но при выводе как обычный текст форматирование теряется или отображается как «сырой» markdown.

## Решение
Для reasoning-режима добавить отдельный слой рендеринга:

- Пытаться отрисовать markdown через внешние CLI-рендеры в приоритете `glow`, затем `mdcat`.
- Если рендереры недоступны или завершились ошибкой, делать fallback на системный `cat`.
- Логику ограничить только `-r` режимом, не меняя стандартный поток выбора/выполнения команд.

## Последствия

- Плюсы: markdown-ответы читаются заметно лучше в терминале.
- Плюсы: поведение остается совместимым в минимальном окружении за счет fallback.
- Минусы: для полноценного рендера нужен установленный внешний инструмент (`glow` или `mdcat`).
