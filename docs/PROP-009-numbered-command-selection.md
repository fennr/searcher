# PROP-009: Выбор команды по номеру в стандартном режиме

## Статус
Принято

## Контекст
Текущий поток подтверждения в стандартном режиме оказался сложным и нестабильным в UX: пользователи видят дублирование вывода и получают неверное ожидание относительно фактического запуска команды.

## Решение
Упростить стандартный режим до детерминированного сценария:

- Получать reasoning-ответ с несколькими вариантами команд.
- Извлекать все валидные shell-команды из ответа модели.
- Показывать пользователю нумерованный список найденных команд.
- Выполнять команду, выбранную по номеру, без дополнительного шага `y/N`.
- При `--dry-run` не выполнять выбранную команду, а только печатать ее.

## Последствия

- Плюсы: предсказуемый UX без дублирования и неоднозначного подтверждения.
- Плюсы: пользователь явно контролирует, какой вариант запускать.
- Минусы: требуется парсинг reasoning-ответа и обработка некорректного ввода номера.
