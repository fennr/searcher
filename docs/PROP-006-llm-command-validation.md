# PROP-006: Дополнительная LLM-валидация команды

## Статус
Принято

## Контекст
Локальная проверка команды отсекает большую часть ошибок, но не всегда покрывает семантическую валидность для конкретного терминального сценария.

## Решение
Добавить опциональный шаг в пайплайн командного режима:

- Флаг `--llm-validate` включает дополнительный запрос к модели.
- Модель возвращает только `VALID` или `INVALID: <причина>`.
- Если результат `INVALID`, утилита пытается один раз отремонтировать команду и повторно валидирует ее.
- При повторном `INVALID` утилита завершает выполнение с ошибкой.

## Последствия

- Плюсы: выше надежность перед исполнением команды.
- Плюсы: хорошо подходит для очень быстрых локальных моделей.
- Минусы: дополнительный API-запрос в обычном режиме при включенном флаге.
